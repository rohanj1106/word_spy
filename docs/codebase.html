1 Project Overview

Trove is a word puzzle game inspired by Word Trip / Wordscapes. Players swipe letters to form words, unlock levels sequentially, earn coins, and listen to ambient music while playing.

graph LR A[📱 iPhone App] --> B[Intro Story] B --> C[Home Screen] C --> D[Adventure Map] C --> E[Daily Challenge] D --> F[Game Screen] F --> G[Letter Wheel] F --> H[Crossword Grid] F --> I[Level Complete] I --> D

🎯 Game Loop

Swipe letters → form words → earn coins → complete all required words → unlock next level

💾 Data Storage

All puzzles in JSON files. User progress (coins, completed levels) in SharedPreferences on-device.

🎨 Design

Custom painted landscape background, Poppins typography, deep indigo + warm gold colour palette.

🔊 Audio

Looping ambient music during gameplay. Coin sound effect on each word found.

3 Flutter — The UI Framework

Flutter is Google's cross-platform UI toolkit. Write once, run on iOS, Android, Web, Mac, Windows. Trove targets iOS + Android + Web.

Core idea: Everything is a Widget — buttons, text, padding, animations, even the app itself. Widgets nest to form a tree. Flutter redraws the tree when state changes.

CustomPainter — The Background

The landscape background (sky, sun, clouds, mountains, trees) is drawn with CustomPainter — like an HTML5 Canvas. Flutter gives you a raw Canvas object and you draw primitives directly.

class _LandscapePainter extends CustomPainter {
  @override
  void paint(Canvas canvas, Size size) {
    // Draw sky gradient
    canvas.drawRect(skyRect, skyPaint);
    // Draw sun
    canvas.drawCircle(sunCenter, 20, sunPaint);
    // Draw mountains, hills, trees...
  }
  @override
  bool shouldRepaint(_LandscapePainter old) => false; // static, never repaint
}

4 Packages (Dependencies)

Packages are libraries added to pubspec.yaml — Flutter's equivalent of package.json.

State

flutter_riverpod ^2.5.0

State management. Providers hold state globally; widgets watch and rebuild automatically when state changes. Like React Context + hooks, but more powerful.

// Define

  (ref) => CoinsNotifier()
);
// Use in widget
final coins = ref.watch(coinsProvider);
Navigation

URL-based navigation. Defines routes like a web router. Handles deep links, back button, path parameters (/game/:puzzleId).

Key-value storage on the device. Like localStorage in the browser. Used to persist coins, completed levels, streak, accessibility settings.

7 State Management — Riverpod

Riverpod is like a global store. Any widget anywhere in the app can read or watch any provider. When state changes, only the widgets that watch that provider rebuild.

sequenceDiagram participant GS as GameScreen participant P as userProgressProvider participant N as UserProgressNotifier participant SP as SharedPreferences GS->>P: ref.watch(userProgressProvider) P-->>GS: UserProgress(coins: 50) GS->>N: ref.read(...).addCoins(10) N->>SP: prefs.setInt('coins', 60) N->>N: state = state.copyWith(coins: 60) N-->>GS: rebuild triggered GS-->>GS: shows coins: 60

Two Types of Usage

// WATCH — rebuilds widget when value changes
final progress = ref.watch(userProgressProvider);
Text('${progress.coins} coins')  // updates automatically

// READ — one-time action, doesn't rebuild
ref.read(userProgressProvider.notifier).addCoins(10);

9 Screens Explained

IntroStoryScreen

4-page cinematic sequence. Uses AnimationController with repeat() for twinkling stars. Each page fades in text with FadeTransition + SlideTransition. A Timer auto-advances pages every 4 seconds. Last page shows the TROVE logo with a gold ShaderMask and a "Begin Adventure" button that navigates to /.

HomeScreen

Shows the app logo, live coin count (from userProgressProvider), a daily challenge card with today's date, and the Play button that goes to /levels. The coin badge rebuilds automatically whenever coins change.

LevelSelectScreen — Adventure Map

Uses LayoutBuilder to get screen width, then calculates node positions in a snake pattern using x-fractions [0.72, 0.50, 0.28, 0.50]. A CustomPainter draws the path between nodes — solid green for completed segments, dashed white for locked ones (using PathMetrics to extract dash segments). Level nodes are StatefulWidgets with AnimationController for the current level's pulsing glow. SingleChildScrollView(reverse: true) means level 1 appears at the bottom on load.

GameScreen

The main gameplay screen. Loads puzzle by ID from PuzzleService. Starts background music on initState. Tracks found words in a Set<String>. On each word: adds coins, plays coin sound, checks if all required words found → triggers level complete. The coin pop animation uses Timer.periodic to move _CoinPop objects upward and fade them out over 1.2 seconds.

LetterWheel

Swipe gesture recognition using GestureDetector onPanStart/onPanUpdate/onPanEnd. Each letter has a GlobalKey. On pan update, RenderBox.globalToLocal() converts finger position to wheel coordinates, then checks distance from each letter center. Selected letters are highlighted with AnimatedContainer. A CustomPaint overlay draws connection lines between selected letters.

11 Full Game Flow

App Launch
main() runs. Sets portrait orientation, transparent status bar. Always routes to /intro. main.dart → TroveApp(initialRoute: '/intro')
Intro Story
4-page cinematic with stars, darkness lifting, TROVE logo. Auto-advances every 4s or user taps. IntroStoryScreen → context.go('/')
Home Screen
Shows coin count from Riverpod, daily challenge card, Play button. HomeScreen → context.push('/levels')
Adventure Map
Loads all puzzles. Draws snake path. Level 1 always unlocked. Others require previous completion. Current level pulses. LevelSelectScreen → context.push('/game/p001')
Game Screen
Loads puzzle JSON. Starts bg music. Player swipes letters to form words. _audioPlayer.play(bg_music.mp3)
Word Found
Word matches required (+10 coins) or bonus (+5 coins). Coin pop animation + coin sound. Progress counter updates. _awardCoins() → addCoins() → SharedPreferences
Level Complete
All required words found. markPuzzleComplete() saves to SharedPreferences. Overlay animates in. _LevelCompleteOverlay → context.go('/levels')
Map Updated
Adventure map reloads. Completed level is now green. Next level is unlocked and pulsing. completedIds now includes p001 → p002 unlocked
Data persistence: Coins and completed levels survive app restarts because they're saved to SharedPreferences (on-device storage) every time they change.
